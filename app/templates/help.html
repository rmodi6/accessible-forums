Welcome to the Accessible Forum

This is a fully accessible forum-like website designed specifically for the visually impaired people. For the best experience, do not use a mouse and use only the keyboard shortcuts mentioned below along with a screen reader like NVDA. If on a mobile device, use the appropriate accessibility tool based on your OS: Talkback on Android and VoiceOver on iOS. Start by searching a keyword like nvda, jaws, skype, etc. in the search field in the navigation bar.

This website uses the Treeview design pattern as specified in the Web Accessibility Initiative (WAI) ARIA authoring practices by W3C org for easy navigation using keyboard shortcuts only. Tested on Mozilla Firefox, Google Chrome and Microsoft Edge browsers using the NVDA screen reader on Windows, VoiceOver on macOS and Talkback on Android.

Note: The nodes mentioned on this page refer to posts in the thread view and threads in the search view.

Keyboard shortcuts

The following keyboard shortcuts are supported by this site:

Keyboard Support
Key Function
↓
  • Moves focus to the next node that is focusable without opening or closing a node.
  • If focus is on the last node, does nothing.
↑
  • Moves focus to the previous node that is focusable without opening or closing a node.
  • If focus is on the first node, does nothing.
→
  • When focus is on a closed node, opens the node; focus does not move.
  • When focus is on a open node, moves focus to the first child node.
  • When focus is on an end node, does nothing.
←
  • When focus is on an open node, closes the node.
  • When focus is on a child node that is also either an end node or a closed node, moves focus to its parent node.
  • When focus is on a root node that is also either an end node or a closed node, does nothing.
Home or Space
  • Moves focus to first node without opening or closing a node.
End
  • Moves focus to the last node that can be focused without expanding any nodes that are closed.
a-z, A-Z
  • Focus moves to the next node with a name that starts with the typed character.
  • Search wraps to first node if a matching name is not found among the nodes that follow the focused node.
  • Search ignores nodes that are descendants of closed nodes.
* (asterisk)
  • Expands all closed sibling nodes that are at the same level as the focused node.
  • Focus does not move.
? (question mark)
  • Focus moves to the highlighted node which is part of the search result within the thread.
  • Expands all the nodes in the path to the focused node.

Views support

This forum has support for three different views: tree, slim and linear. It can be changed by changing the v query param in the url.

Views Support
View Description
tree (default) Shows the nodes in a tree like view with a parent child kind of relationship between them.
slim Similar to the tree view but with smaller depth of nodes.
linear Shows all the nodes in a linear fashion with no relationship between the nodes.
